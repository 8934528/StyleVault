# StyleVault Security and Acknowledgement

## Purpose

This project is built explicitly for educational purposes and game development logic practice. It aims to demonstrate proficiency in building modern web applications, integrating full-stack functionality using ASP.NET Core, React, and structured probability-based logic patterns safely.

## Acknowledgment of Gaming Rules

The project simulates a "lucky draw" or "pack opening" mechanic. By interacting with this codebase or compiled software, you acknowledge the following rules:

- **Fake Tokens and Money**: All representations of currency ("ZAR", "Rands", "Coins", "Tokens") are 100% fictional. They hold absolutely no real-world value and cannot be exchanged.

- **Entertainment Only**: The probability and mechanic implementations are designed solely for fun and entertainment. The game is deterministic based on a controlled fake RTP (Return to Player) model and should not be confused with or used as real gambling infrastructure.

- **No Real Transactions**: There are no real payment gateways connected. When users "purchase" packs or "top up" their balance, it is merely simulating the action via state management and mock numbers.

Have fun parsing the logic and enjoying the virtual drops!
